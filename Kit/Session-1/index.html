What is Node.js ?

  • A platform built on Chrome's JavaScript runtime .
  • Uses an event-driven, non-blocking I/O model.
  • Written in C/C++ & JavaScript & its apps are written in JavaScript.
  • Applications are run as single-threaded.
  • Build-in asynchronous I/O, file-system, socket ,  HTTP & cluster .
  • Acts as a web server.
  • It provides the event loop as a language instead of a library.
  • The ability to write JavaScript on the server.

What is NPM ?

  • NPM is the official package manager for Node.js.
  • Runs through the command line and manages dependencies.
  • Allows users to install Node.js applications.
  • NPM is written entirely in JavaScript, and runs on the Node.js platform.

History

  • Created by Ryan Dahl and was first  published in 2009, could only run on Linux.
  • Ryan was inspired by seeing a file upload progress bar on Flickr.
  • Sponsored by Joyent.
  • Originally Ryan had several failed projects in C and Haskell, but when Google's V8 engine was released, it was success.
  • In June 2011, Microsoft partnered with Joyent to help create a native Windows version of Node.js
  • NPM, a package manager for Node.js libraries, was introduced in 2011.

Design


  • Ryan chose JavaScript because of the lack of an existing I/O API. This allowed him to define a convention of non-blocking, event-driven I/O.
  • Node.js is part of the Server Side JavaScript environment and extends the JavaScript API to offer server side features.
  • It uses an event loop, instead of processes or threads.

Runs JavaScript on the server.

server = ws.listen(3000);

    socket.on('message', function (data) { 
    console.log('message received:', data);
    });
    
    socket.on('close', function () { 
     console.log('socket closed!');

Unlike most JavaScript programs, Node.js is not executed in a web browser, but instead as a server-side JavaScript application.

Architecture



benefits

  • Savings in I/O cost (i.e., high performance)
  • Savings in Memory.
  • End-2-End JavaScript/Coffee-Script.
  • Thousands of libraries. Search at npmjs.org
  • Second most popular watched project on Github.
  • Build CLI based tools such as Grunt or Make.
  • Build high-performance, high I/O, TCP apps like proxy servers, PaaS, databases, etc.
  • Build a (soft) real-time social app like Twitter or a chat app.
  • Add a RESTful API-based web server in front of an application server.

Drawbacks

  • Libraries are still in developed.
  • Many libraries, such as the SAML auth library which is required for enterprise apps, are in development available yet.
  • Learning curve burden to server-side programmers.
  • Asynchronous and event-driven code inherently adds more complexity to the code versus a synchronous code.
