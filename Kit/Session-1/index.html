What is Node.js ?

  • An open source  cross-platform built on Chrome's JavaScript runtime  environment.


  • Uses an event-driven, non-blocking I/O model.

  • Written in C/C++ & JavaScript & its apps are written in JavaScript.

  • Used commonly for real-time web apps.


  • Applications are run as  single-threaded.

  • Build-in  asynchronous I/O , file-system ,  socket ,   HTTP  &  cluster .

  • Acts as a  web server.

  • The ability to write JavaScript on the server.

    • History

    • Created by Ryan Dahl and first  published in 2009, could only run on Linux.
    • Sponsored by Joyent .
    • June 2011, Microsoft partnered with Joyent to help create a native Windows version of Node.js.
    • Janunary 2012, Dahl stepped aside, npm creator Isaac Schlueter manage the project.
    • January 2014, Timothy J. Fontaine became new project lead.
    • December 2014, Fedor Indutny started IO.js

    Architecture

    benefits

    • Savings in I/O cost (i.e., high performance)
    • Savings in Memory.
    • End-2-End JavaScript/Coffee-Script.
    • Thousands of libraries. Search at npmjs.org
    • Second most popular watched project on Github.
    • Build CLI based tools such as Grunt or Make.
    • Build high-performance, high I/O, TCP apps like proxy servers, PaaS, databases, etc.
    • Build a (soft) real-time social app like Twitter or a chat app.
    • Add a RESTful API-based web server in front of an application server.

    Drawbacks

    • Libraries are still in developed.
    • Many libraries, such as the SAML auth library which is required for enterprise apps, are in development available yet.
    • Learning curve burden to server-side programmers.
    • Asynchronous and event-driven code inherently adds more complexity to the code versus a synchronous code.
