What is Node.js ?

  •  A platform built on Chrome's JavaScript runtime.
  •  Uses an event-drive, non-blocking I/O model.
  •  Written in C/C++ & JavaScript. And its applications are written in JavaScript.
  •  Applications are run as single-threaded.
  •  Build-in asynchronous I/O, file-system, socket and HTTP.
  •  Acts as a web server.
  •   It provides the event loop as a language instead of a library.

History

  •  Created by Ryan Dahl and was first  published in 2009, could only run on Linux.
  •  Ryan was inspired by seeing a file upload progress bar on Flickr.
  •   Sponsored by Joyent.
  •  Originally Ryan had several failed projects in C and Haskell, but when Google's V8 engine was released, it was success.
  •  In June 2011, Microsoft partnered with Joyent to help create a native Windows version of Node.js
  •  NPM, a package manager for Node.js libraries, was introduced in 2011.

Design

  •  The goal was push capabilities such as those seen in Gmail. 
  •  Ryan chose JavaScript because of the lack of an existing I/O API. This allowed him to define a convention of non-blocking, event-driven I/O.
  •  Node.js is part of the Server Side JavaScript environment and extends the JavaScript API to offer server side features.
  •  It uses an event loop, instead of processes or threads.


Similar environments

  • Tornado and Twisted for Python.
  • Perl Object Environment for Perl .
  • Vert.x for Java.
  • EventMachine for Ruby .


Unlike most JavaScript programs, Node.js is not executed in a web browser, but instead as a server-side JavaScript application.

Architecture

Advantages

  • Node.js allows the reuse of the same model and service interface between client-side and server-side.
  • The fact that JavaScript had no unified API for IO allowed the developers of Node to think about the best way to implement a modern IO interface.
  • Implement all IO in an asynchronous and non-blocking way, combined with a single-threaded event-based loop, presented a novel way to implement real-time web applications.


  • Node.js can therefore keep many connections alive transparently without having to reject new incoming connections.
  • Node.js is a lightweight platform, a step away from the heavier frameworks and infrastructures.

Drawbacks

  • Node.js libraries are developed actively, with a high rate of change from month-to-month. This can cause version issues, instabilities and certain development issues.


  • Npm shrink wrap and package.json were introduced a while back to set up standards, but the issue of standards is still a work in process and some issues still exist.
  • The whole callback, event-driven, functional programming aspects of Node.js can add a learning curve burden to server-side programmers of other object-oriented languages.
  • Asynchronous and event-driven code inherently adds more complexity to the code versus a synchronous code.
  • As a JavaScript derivative, Node.js code is loosely-typed with Object-orientation abilities. This combination can result in code that is hard to debug.
  • JavaScript has more than its share of "bad parts" that might throw off engineers and newcomers.
